Vscode Plugin开发入门探索

发表于 2019-12-22 | 阅读次数:

简单分了三块，开发准备、注册事件、贡献点、插件开发比预想的简单很多，官方文档很详细，主要还是在于创造力

准备

安装vscode 官方专用Plugin脚手架，推荐使用Typescript进行开发

然后只要了解下面介绍的三个概念就能按照自己idea很快地制作一款自己的专属插件

注册事件

Activation Events: events upon which your extension becomes active.

事件写在插件所在项目的package.json中，后面可以发现一款插件package.json文件中定义了非常多的内容

  • onLanguage
  • onCommand
  • onDebug
  • onDebugInitialConfigurations
  • onDebugResolve
  • workspaceContains
  • onFileSystem
  • onView
  • onUri
  • onWebviewPanel
  • *

以上是Version1.41中支持的响应事件，更具体参考文档

贡献点

Contribution Points: static declarations that you make in the package.json Extension Manifest to extend VS Code.

这也是一个声明在Package.json内的配置，可以理解为当前这款插件主要作用在Vscode的哪一部分，目前版本支持以下定制

  • configuration
  • configurationDefaults
  • commands
  • menus
  • keybindings
  • languages
  • debuggers
  • breakpoints
  • grammars
  • themes
  • snippets
  • jsonValidation
  • views
  • viewsContainers
  • problemMatchers
  • problemPatterns
  • taskDefinitions
  • colors
  • typescriptServerPlugins
  • resourceLabelFormatters
  • contributes.configura

还是以demo为例，我们需要手动调用refresh命令，光在activationEvents里注册事件是不够的，这里只是告诉vscode这个命令可以用来激活，你可以在contribution points里声明上十个命令，但是只有在activationEvents注册过的命令才是会生效的，另外，在contribution points中声明的命令是会出现在命令面板里的

插件只有 activate 和 deactivate 两种状态，所以当在contribution points中声明好命令，在activationEvents中注册好之后，只要通过Vscode API 在 extension.ts 中完善好事件处理就行了
